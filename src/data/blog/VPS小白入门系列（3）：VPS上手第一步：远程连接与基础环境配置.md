## 一、前言：远程连接=“隔空操控”你的VPS
在上一篇，我们已经选好并买到了适合自己的VPS，现在就进入最关键的实操环节——远程连接。

简单说，**远程连接就是通过你本地的电脑，隔空控制远程机房的VPS**，就像坐在VPS的“屏幕前”操作一样，能敲命令、装软件、搭服务。

新手不用慌，远程连接没有想象中复杂，全程跟着步骤点鼠标、敲命令就行，而且我们会覆盖「Windows/Mac/Linux」所有系统，不管你用什么电脑，都能轻松连接。

先提前准备好3个核心信息（买VPS后，商家会通过邮件/控制台发给你），缺一不可：
1.  VPS的 **公网IP地址**（一串数字，如：123.45.67.89）
2.  登录账号（默认一般是：root，最高权限，小白先用地狱模式）
3.  登录密码（买VPS时设置的，或商家默认密码，记得修改）

## 二、远程连接工具：按系统选，小白优先可视化工具
不同系统对应不同的远程连接工具，我们分3类讲，每类都推荐「小白友好型工具」，不用记复杂命令，可视化操作更简单。

### 1. Windows系统（最常用，重点讲）
推荐2款工具，任选其一，新手优先选「FinalShell」（可视化强，不用手动输命令）：
#### 工具1：FinalShell（新手首选）
- 优点：免费、可视化界面、操作简单，支持密码/密钥登录，还能直接上传文件，一站式搞定
- 操作步骤（手把手）：
  1.  下载安装：百度搜索「FinalShell」，进入官网下载，一路下一步安装（默认选项即可）
  2.  打开软件，点击左上角「新建」→ 选择「SSH连接」
  3.  填写连接信息：
      - 名称：随便填（如：我的VPS），方便区分
      - 主机：填写VPS的公网IP地址
      - 端口：默认22（不用改，除非你后续修改了端口）
      - 用户名：root（默认，不用改）
      - 密码：填写VPS的登录密码
  4.  点击「确定」，然后双击左侧的连接名称，弹出提示点「接受」，就能成功连接VPS啦（连接成功后，下方会出现黑色命令行窗口）

#### 工具2：Xshell（备选，专业稳定）
- 优点：专业、稳定，适合长期使用，支持多窗口切换
- 操作步骤（简化版）：
  1.  下载安装Xshell（个人版免费），打开后点击「新建会话」
  2.  主机填写VPS公网IP，端口22，协议SSH，点击「确定」
  3.  双击会话，输入用户名root，再输入密码（输入密码时不显示，输完按回车），即可连接

### 2. Mac系统
不用额外下载工具，用系统自带的「终端」就能连接，也可以用FinalShell（和Windows操作一致）：
#### 系统自带终端（无需下载）
1.  打开「终端」（启动台→其他→终端）
2.  输入命令：`ssh root@你的VPSIP`（如：ssh root@123.45.67.89），按回车
3.  弹出提示「Are you sure...」，输入yes，按回车
4.  输入VPS登录密码（输完不显示，按回车），即可成功连接

### 3. Linux系统
和Mac一样，自带终端，直接输入命令连接：
1.  打开终端，输入命令：`ssh root@你的VPSIP`，按回车
2.  输入yes确认，再输入密码，按回车，即可连接

### 小白避坑：连接失败怎么办？
最常见的3个问题，直接对应解决，不用慌：
1.  提示「Connection refused」（连接被拒绝）：检查VPS的22端口是否开放，或VPS是否正常开机（去商家控制台重启VPS）
2.  提示「Permission denied」（权限拒绝）：用户名输错（默认是root），或密码输错（重新输入，注意大小写）
3.  无法连接、超时：检查本地网络是否正常，或VPS线路是否有问题（换个网络试试，如手机热点）

## 三、两种登录方式：密码登录（新手首选）vs 密钥登录（更安全）
远程连接有两种方式，新手先用地狱模式，熟悉后再切换到密钥登录（更安全，避免密码被暴力破解）。

### 1. 密码登录（当前正在用，新手首选）
就是我们上面操作的方式，输入root账号+密码登录，优点是简单、不用额外配置，缺点是密码可能被暴力破解，适合新手初期使用。

### 2. 密钥登录（进阶安全，推荐后续配置）
不用输密码，通过「密钥文件」登录，安全性极高，小白跟着步骤也能配置：
#### 操作步骤（以Windows FinalShell为例）：
1.  打开FinalShell，点击左侧连接名称，右键→「属性」
2.  切换到「密钥登录」，点击「生成密钥」，默认选项，点击「生成」
3.  生成后，点击「保存私钥」（保存到本地电脑，记住路径，别弄丢）
4.  点击「上传公钥到服务器」，自动上传密钥到VPS，提示成功后，点击「确定」
5.  断开当前连接，重新双击连接，无需输密码，直接就能登录

#### 补充：Mac/Linux生成密钥（简化版）
1.  终端输入命令：`ssh-keygen`，按3次回车（默认生成，不用设置密码）
2.  输入命令：`ssh-copy-id root@你的VPSIP`，输入VPS密码，即可上传公钥
3.  下次连接，直接输入`ssh root@你的VPSIP`，无需输密码

## 四、首次连接VPS：必做5件事，安全又流畅
连接成功后，别着急搭服务，先做这5件基础配置，能避免后续很多麻烦，让你的VPS更安全、运行更流畅，全程复制命令即可，不用懂含义。

### 1. 修改默认密码（重中之重，防止被破解）
默认密码可能被商家泄露，或太简单被暴力破解，第一步必须修改：
1.  在命令行输入命令：`passwd`，按回车
2.  输入「新密码」（输完不显示，建议设复杂一点，字母+数字+符号），按回车
3.  再次输入新密码，按回车，提示「successfully」就是修改成功

### 2. 更新系统软件包（修复漏洞，获取最新版本）
VPS默认系统可能有旧漏洞，更新后更安全，不同系统命令不同，二选一：
- Ubuntu系统（新手首选）：输入命令 `apt update && apt upgrade -y`，按回车，等待更新完成（出现「done」即可）
- CentOS系统：输入命令 `yum update -y`，按回车，等待更新完成

### 3. 关闭无用服务（节省资源，提高稳定性）
VPS默认会开启一些无用服务，占用内存和CPU，新手直接执行命令关闭：
```bash
# 关闭无用服务（通用命令，两种系统都能用）
systemctl stop postfix
systemctl disable postfix
```
输入后，没有报错就是成功，不用管具体含义。

### 4. 配置防火墙（只开放需要的端口，防攻击）
防火墙能阻止非法访问，小白只需开放「22端口（远程连接）」，后续搭服务再开放对应端口：
#### Ubuntu系统：
```bash
# 安装防火墙
apt install ufw -y
# 开放22端口（远程连接必备）
ufw allow 22
# 启动防火墙
ufw enable
# 查看防火墙状态
ufw status
```
#### CentOS系统：
```bash
# 启动防火墙
systemctl start firewalld
# 开放22端口
firewall-cmd --permanent --add-port=22/tcp
# 重新加载防火墙
firewall-cmd --reload
# 查看状态
firewall-cmd --list-ports
```

### 5. 安装基础工具（后续搭服务会用到）
安装一些常用工具，避免后续重复安装，直接复制命令执行：
- Ubuntu系统：`apt install -y wget curl vim`
- CentOS系统：`yum install -y wget curl vim`

## 五、基础环境配置：安装Docker（复用之前知识点，后续实战必备）
我们后续搭博客、云盘、导航站，都会用到Docker，现在提前安装好，复用之前Docker系列的知识点，小白直接复制命令即可：

### 一键安装Docker（通用命令，Ubuntu/CentOS都能用）
1.  输入命令（复制整行，按回车）：
```bash
curl -fsSL https://get.docker.com | sh
```
2.  等待安装完成，出现「Docker CE has been installed」就是成功
3.  启动Docker，并设置开机自启（避免VPS重启后Docker失效）：
```bash
systemctl start docker
systemctl enable docker
```
4.  验证是否安装成功：输入 `docker --version`，能显示Docker版本号，就是安装成功

### 小白避坑：Docker安装失败怎么办？
- 提示「curl: command not found」：先安装curl（Ubuntu：apt install curl -y；CentOS：yum install curl -y）
- 安装后无法启动：重启VPS（命令：`reboot`），再重新启动Docker

## 六、实操总结：远程连接+基础配置步骤回顾
怕新手记混，整理了「一步到位实操流程」，直接抄作业：
1.  准备VPS的IP、root账号、密码
2.  按自己的系统，选择工具连接VPS（Windows用FinalShell，Mac/Linux用终端）
3.  连接成功后，修改默认密码（passwd命令）
4.  更新系统软件包（apt/yum update命令）
5.  关闭无用服务，配置防火墙
6.  安装基础工具和Docker
7.  （可选）配置密钥登录，提高安全性
